Palette

Slate background, muted electric blue accent, ok/warn/err as narrow exceptions. No gradients except the subtle titlebar shimmer.

Type

Inter 14/500 body, 17/600 section titles, -0.01em tracking. JetBrains Mono 11–12 for timestamps, file meta, log, labels.

Structure

Fixed 2-column: left 400px (input + languages + submit), right flexes (progress · result · error · decision). Never resizes.

Behavior

Right column swaps its body per state. Stage bar + log appear only when the pipeline is actually running on the server.

Coding handoff — what to lift

  • CSS tokens at the top of this file → drop straight into tokens.css
  • 6 states = 6 values of a pipelineState enum → switch in right column body
  • Stage bar is a single component fed by an array of {name, status, duration} — status ∈ done | active | pending | fail
  • Log is append-only with 3 severity classes: .i / .w / .e, plus .hdr for stage headers and .dim for meta lines
  • Left column never changes structure — only disabled states and submit-button label/variant change

State transition rules

  • idle → uploading on submit with file
  • uploading → running on 200 from POST · or → rejoin on 409
  • running → succeeded on SSE job.complete
  • running → failed on SSE job.error
  • rejoin → running when user clicks "Rejoin" (SSE subscribes to the other job)
  • succeeded / failed → idle on "Submit another" or "delete job"
